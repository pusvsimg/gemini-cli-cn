第一部分：安装与初次启动

1. 环境准备

在安装前，请确保您的系统已安装 Node.js (推荐 v20 或更高版本) 和 npm。您可以在终端运行以下命令来检查：

node -v
npm -v

2. 全局安装

打开您的终端，运行以下命令进行全局安装：

npm install -g @google/gemini-cli

3. 首次运行与认证

  1. 安装成功后，在终端输入 gemini 并按回车
  2. 首次运行时，它会引导您完成认证
  3. 选择认证方式，推荐 "Login with Google"
  4. 您的浏览器将自动打开一个 Google 登录和授权页面
  5. 完成授权后，返回终端，您就可以开始与 Gemini 对话了

第二部分：核心交互模式

在深入了解具体的 / 命令之前，您需要掌握与 Gemini-CLI 交互的三种基本方式：

1. 自然语言提示

这是最基础的交互方式。直接输入您的问题或指令，就像聊天一样。

请帮我用 Python 写一个快速排序算法。

2. @ 命令：文件/目录上下文注入

使用 @ 符号，可以将本地文件或整个目录的内容作为上下文提供给 Gemini。这是其理解项目背景的关键。

请根据 @./src/utils.js 文件中的代码风格，为这个新函数添加注释。

3. ! 命令：直接执行 Shell 命令

使用 ! 符号，可以直接在 CLI 内部执行任何终端命令，无需退出。

!git status

第三部分：斜杠命令终极解析

斜杠 (/) 命令是控制和管理 Gemini-CLI 的核心。下面是对每个命令的详细剖析。

会话与历史管理

/chat save 保存当前完整的对话状态
/chat resume 恢复一个之前保存的对话
/chat list 列出所有已保存的会话标签
/chat clear 清空当前对话的历史记录
/history 查看当前会话的详细历史记录
/compress 将冗长的对话历史压缩成简洁摘要

上下文与记忆管理

/memory show 显示当前所有已加载的 GEMINI.md 文件内容
/memory refresh 重新加载所有记忆文件
/context show 显示即将发送给模型的所有信息
/init 分析项目并创建 GEMINI.md 文件

工具与集成

/tools 列出所有可用的工具及其状态
/extensions 列出当前活动的扩展
/setup-github 配置 GitHub Actions 工作流
/yolo 启用自动执行模式（谨慎使用）

配置与信息

/help 显示所有可用的斜杠命令
/settings 打开全局配置文件
/theme 更改 CLI 界面的颜色主题
/stats 显示当前会话的详细统计数据

第四部分：高级技巧与工作流

1. 代码调试工作流

  1. 使用 @/path/to/file.py 载入问题代码
  2. 提问："分析这段代码可能出现的 IndexError 异常，并给出修复建议"
  3. 使用 !pytest tests/ 运行相关测试
  4. 让 Gemini 直接生成修复后的代码，并使用 /copy 粘贴

2. 自动化文档生成

  1. 在项目根目录创建 GEMINI.md，写入指令："所有文档需为 Markdown 格式，包含代码示例"
  2. 运行 /memory refresh 加载指令
  3. 提问："请为 @./src/ 目录下的所有模块生成 API 文档"

第五部分：深度自定义

1. 配置 settings.json

  1. 运行 /settings 找到并打开配置文件
  2. 修改默认模型："model": "gemini-1.5-pro-latest"
  3. 调整创造性："temperature": 0.8
  4. 禁用危险工具："tools": { "shell": { "execute": false } }

2. 创建自定义斜杠命令

  1. 找到命令目录（通常是 ~/.gemini/commands）
  2. 创建名为 deploy 的可执行脚本文件
  3. 添加执行权限：chmod +x ~/.gemini/commands/deploy
  4. 重启 Gemini-CLI，现在可以直接运行 /deploy
#!/bin/bash
echo "🚀 开始部署项目..."
# 在此添加您的部署命令
echo "✅ 部署完成！"

第六部分：常见问题解答

Q: 安装时出现权限错误怎么办？

A: 尝试修复 npm 的全局包权限，避免直接使用 sudo。可以搜索 "npm fix permissions" 查找解决方案。

Q: Gemini 似乎没有理解我的项目文件？

A: 确认您使用了 @ 命令正确注入了文件或目录。使用 /context show 检查当前上下文是否包含了您期望的文件内容。

Q: 如何重置认证信息？

A: 运行 /auth 命令，里面通常会提供重置或重新登录的选项。
